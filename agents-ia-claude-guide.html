Guide pratique · Claude Code & Agents

Créer une équipe d'agents IA dans Claude qui travaillent à votre place

Comment architecturer, orchestrer et optimiser des agents autonomes dans Claude Code — pour tout professionnel, quel que soit son métier.

Contexte

Ce guide s'appuie sur un cas réel : la construction d'une équipe marketing complète (4 agents + 1 orchestrateur) autour d'une marque fictive. Les principes sont transposables à n'importe quel domaine — RH, finance, juridique, commercial, formation.

01

Poser les fondations du projet

Avant de créer le moindre agent, il faut constituer une base de connaissance structurée dans un dossier dédié. C'est la matière première que tous les agents partageront.

1
Créer l'arborescence du projet

Un dossier racine avec des sous-dossiers clairs : contexte/, pos/, exemples/, data/, trimestre/, réseau/. Tous vides au départ — ils seront remplis progressivement.

2
Générer les fichiers de contexte (.md)

Demander à Claude de produire 3 à 4 fichiers Markdown documentant votre sujet : historique, produits/services, style de communication, stratégie. Plus ces fichiers sont détaillés, plus les agents seront précis.

3
Télécharger et placer dans contexte/

Ces fichiers servent de mémoire partagée à toute l'équipe d'agents. Ils sont lus automatiquement à chaque initialisation.

Astuce tokens

Des fichiers de contexte bien structurés réduisent la répétition dans chaque prompt. L'agent n'a pas besoin qu'on lui réexplique le contexte à chaque fois : il le lit une fois dans CLAUDE.md.

02

Le cerveau du projet : CLAUDE.md

Le fichier CLAUDE.md est le point névralgique de tout projet Claude Code. C'est la mémoire persistante du projet : Claude le lit systématiquement au lancement et y retrouve les règles, la structure, le ton, les outils disponibles.

Commande d'initialisation
# Dans Claude Code, première action à faire : Initialise un fichier CLAUDE.md pour ce projet. # Claude lit tous les fichiers du dossier et génère # un document racine synthétique automatiquement.

Le fichier généré contiendra typiquement :

🗂️
Structure du répertoire

L'arborescence des dossiers et leur rôle.

🎯
Mission & règles

Les contraintes à respecter (ton, langue, format de sortie).

🔧
Skills disponibles

La liste des agents/compétences activables via /.

✏️
Modifiable à tout moment

Vous pouvez éditer ce fichier manuellement pour enrichir ou corriger le contexte.

Principe clé

Un bon CLAUDE.md évite de répéter le contexte dans chaque prompt. C'est l'équivalent d'un brief permanent que l'agent consulte avant chaque tâche — une économie massive de tokens sur la durée.

03

Les quatre agents spécialisés

Chaque agent est un skill : un fichier Markdown structuré qui définit les capacités, les étapes et les outils d'un agent. Une fois créé, il s'appelle avec /nom-du-skill.

🔍
Agent 1 · Recherche

Analyse de marché, veille concurrentielle, identification des tendances. Connecté à Perplexity via MCP pour des données en temps réel.

✍️
Agent 2 · Contenu

Création de posts pour les réseaux sociaux, s'appuyant sur l'analyse des meilleurs contenus existants et les frameworks de storytelling.

📊
Agent 3 · Données

Analyse de fichiers Excel de performance, génération de dashboards HTML interactifs, extraction d'insights actionnables.

🖥️
Agent 4 · Présentation

Transformation des livrables en présentations PowerPoint soignées, avec charte graphique intégrée et titres narratifs.

Comment créer un skill agent

Prompt type pour créer un agent
Lis le document POS dans le dossier pos/. Transforme-le en un skill appelé "agent-recherche" capable de faire des analyses de marché. Avant de commencer, s'il existe des fichiers pertinents dans contexte/ ou CLAUDE.md, lis-les. Ce skill doit : - Suivre les étapes définies dans le POS - Utiliser Perplexity (MCP) pour la recherche - Enregistrer les résultats dans data/ - Utiliser le skill-creator pour sa construction
À retenir

Un skill bien rédigé contient : son objectif, ses entrées, ses étapes, ses outils, et le format de sortie attendu. Plus le skill est précis, moins Claude improvise — et moins il consomme de tokens en clarifications inutiles.

Connexion aux outils via MCP

Les agents peuvent se connecter à des services externes via le protocole MCP (Model Context Protocol). Exemples utiles :

Outil MCP Usage typique Type d'agent
Perplexity Recherche web en temps réel, veille concurrentielle Recherche
Nano Banana Génération d'images et visuels Contenu visuel
Google Drive / Slack Récupération et diffusion de fichiers Coordination
CRM / Excel Lecture de données structurées Analyse
04

Orchestrer une équipe d'agents

L'intérêt des agents ne réside pas dans leur utilisation individuelle, mais dans leur collaboration orchestrée. Un 5e agent-orchestrateur pilote les quatre autres, répartit les tâches en parallèle, organise les révisions croisées et consolide les livrables.

Agent 1Recherche & stratégie
→
Revue croiséeAgents 1 & 2 s'échangent les retours
→
Agent 2Analyse données
→
Agent 3Présentation finale

Le prompt d'orchestration

Structure d'un prompt multi-agents
Tu vas créer une Agent Team avec une liste de tâches partagées pour gérer ce projet. # 1. Définir les agents et leurs rôles Mets en place 3 coéquipiers spécialisés : - Agent Stratège : recherche et analyse de marché - Agent Analyste : performance des données de campagne - Agent Présentateur : synthèse en slides # 2. Définir les dépendances et la revue croisée Avant la finalisation : - Agents 1 et 2 relisent mutuellement leurs livrables - Ils signalent les incohérences et divergences - Chacun révise son livrable avant de soumettre à l'Agent 3 # 3. Demander un plan avant exécution Présente-moi le plan d'exécution avant de commencer.
Ce qui se passe concrètement

Les agents produisent des livrables V1, échangent des retours structurés (corrections, signalement d'incohérences), révisent leurs propres outputs, puis soumettent les livrables révisés au dernier agent qui synthétise. C'est un processus éditorial automatisé.

Pourquoi demander un plan avant l'exécution

🎯
Valider l'interprétation

Claude vous montre comment il a compris la tâche avant d'agir. Vous corrigez en amont plutôt qu'après coup.

💰
Économiser des tokens

Une mauvaise interprétation coûte des centaines de tokens. Un plan en 100 tokens vaut mieux qu'un livrable à refaire.

🔍
Repérer les manques

Le plan révèle les fichiers manquants, les ambiguïtés ou les périmètres mal définis avant que le travail ne commence.

05

Packager en plugin réutilisable

Une fois l'équipe d'agents construite, ses compétences peuvent être condensées dans un plugin portable. Ce plugin peut être rechargé dans d'autres contextes, sur d'autres projets, avec d'autres données.

Créer le plugin
Regroupe tous les skills du dossier skill/ dans un plugin appelé "AI-Marketing-Skills". Réfère-toi à la documentation officielle pour créer les skills dans le bon format. Mets à jour CLAUDE.md pour référencer ce plugin.
1
Générer le plugin (fichier .zip)

Claude regroupe tous les skills en un seul package avec un manifeste et une structure officielle.

2
L'importer dans Cowork

Dans l'interface Cowork de Claude, section "Plugins", téléverser le fichier .zip. Le plugin apparaît et ses skills deviennent invocables.

3
Le réutiliser sur un nouveau projet

Sélectionner les skills nécessaires et appliquer les mêmes compétences à de nouvelles données — sans reconfigurer les agents depuis zéro.

Bénéfice principal

Les agents construits une fois deviennent des compétences portables. La mise en place initiale représente un investissement unique ; chaque réutilisation en amortit le coût — en temps et en tokens.

06

Optimiser la gestion des tokens

Tout ce qui précède n'est pas qu'une architecture d'agents : c'est une stratégie d'efficacité des tokens. Voici les mécanismes en jeu, et comment les optimiser consciemment.

Ce qui consomme des tokens

Source de consommation Impact Levier
Répéter le contexte dans chaque prompt Élevé Centraliser dans CLAUDE.md
Prompts vagues forçant des clarifications Élevé Structurer les prompts avec des étapes
Livrables à refaire suite à une mauvaise interprétation Élevé Demander un plan avant exécution
Instructions redondantes entre agents Moyen Mutualiser dans les skills partagés
Longs historiques de conversation Moyen Démarrer de nouvelles sessions avec CLAUDE.md
Fichiers de contexte bien rédigés (coût initial) Amorti Investissement rentable sur la durée

Les 7 principes d'économie de tokens

📄
1. CLAUDE.md comme mémoire persistante

Toute information qu'on répète souvent mérite d'être dans ce fichier. Une fois écrite, elle n'occupe plus de place dans vos prompts.

🧩
2. Skills au lieu de prompts longs

Un skill stocke un processus complet. Au lieu de 200 tokens d'instructions, vous tapez /nom-skill.

🗺️
3. Plan avant exécution

Toujours demander à Claude de présenter son plan avant d'agir sur une tâche complexe. Correction en amont = économie massive.

📁
4. Références de fichiers plutôt que copier-coller

Pointer vers data/rapport.xlsx au lieu de coller son contenu dans le prompt. Claude lit directement le fichier.

🔄
5. Sessions fraîches pour les nouvelles tâches

Un long historique de conversation pèse sur chaque requête. Démarrer une nouvelle session avec CLAUDE.md repart de zéro proprement.

⚡
6. Parallélisme d'agents

Des agents qui travaillent en parallèle réduisent la longueur de la chaîne. Moins de tours de parole = moins de contexte accumulé.

🏗️
7. Investir dans la structure initiale

Une heure passée à bien rédiger CLAUDE.md et les skills économise des dizaines d'itérations futures. Le coût initial est amorti rapidement.

Synthèse

L'architecture agents n'est pas qu'une question de puissance de traitement : c'est une discipline d'ingénierie du contexte. Structurer l'information en amont, découper les responsabilités, éviter les répétitions — ces pratiques réduisent la consommation de tokens tout en augmentant la qualité des outputs.

07

Récapitulatif opérationnel

Le chemin complet, de zéro à une équipe d'agents fonctionnelle :

1
Créer l'arborescence

Dossier racine + sous-dossiers contexte/, pos/, exemples/, data/, résultats/.

2
Alimenter contexte/

Générer 3 à 4 fichiers .md décrivant votre domaine, vos processus, votre style. Les déposer dans ce dossier.

3
Initialiser CLAUDE.md

Lancer "Initialise un fichier CLAUDE.md pour ce projet". Claude lit tout et crée la mémoire persistante du projet.

4
Créer les skills / agents

Pour chaque agent, fournir un POS ou un guide de processus et demander à Claude de le transformer en skill.

5
Tester chaque agent individuellement

Invoquer avec /nom-skill et valider les outputs avant l'orchestration.

6
Orchestrer en équipe

Utiliser un prompt d'orchestration multi-agents avec revue croisée, demander le plan avant exécution.

7
Packager en plugin

Consolider tous les skills dans un plugin .zip réutilisable sur d'autres projets et dans Cowork.

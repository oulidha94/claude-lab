01

Introduction

Pourquoi changer d'outil ?

Claude Cowork est le parfait entre-deux entre Claude en mode chat (usage conversationnel) et Claude Code (environnement développeur). Il s'adresse aux professionnels qui veulent automatiser des tâches récurrentes sans entrer dans le développement logiciel.

💬

Claude Chat

Usage conversationnel. Idéal pour des questions ponctuelles et des tâches uniques, sans mémoire persistante ni automatisation.

⚙

Claude Cowork recommandé

Pour les tâches récurrentes, les projets avec mémoire, les fichiers locaux et les agents semi-automatisés. L'idéal pour débuter.

👨‍💻

Claude Code

Environnement développeur complet. Agit comme un développeur à part entière. Recommandé pour les profils techniques.

Objectif de ce guide À l'issue de cette session, vous serez capable de configurer un projet Cowork, lancer des tâches programmées, connecter vos outils métier, créer un live artifact et utiliser Claude depuis votre navigateur.
02

Dans quel cas l'utiliser ?

Prérequis et critères de passage

Prérequis techniques

1

Télécharger l'application desktop Claude

Disponible sur Mac et PC depuis le site officiel de Claude (claude.ai). L'application desktop est indispensable ; la version web ne suffit pas.

2

Disposer d'un abonnement payant

La version Pro à 17 $/mois est suffisante pour démarrer avec Cowork et lancer des agents basiques. L'abonnement supérieur (~100 $/mois) n'est utile que pour du développement applicatif intensif.

3

Accéder à la section "Cowork"

Une fois l'application installée, l'interface Cowork est accessible via le menu en haut de l'application. L'interface change visuellement dès l'activation.

Signaux de passage de Chat à Cowork

🔁

Tâches récurrentes

Vous posez les mêmes questions à Claude chaque semaine : analyse de pipeline, rapport hebdomadaire, veille, etc.

🧠

Contexte projet complexe

Votre projet nécessite que Claude retienne un contexte important d'une session à l'autre sans que vous le répétiez.

📁

Fichiers locaux

Vous avez besoin que Claude lise et écrive directement dans des dossiers de votre PC plutôt que via des pièces jointes.

🌐

Navigation web autonome

Vous souhaitez que Claude navigue lui-même sur le web pour récupérer des informations en temps réel.

Bonne pratique Commencez par une seule tâche récurrente pour prendre en main Cowork, avant de construire des workflows plus complexes. La maîtrise du paramétrage d'un projet est le prérequis de tout le reste.
03

Les bases

Projets, mémoire, tâches programmées, skills et connecteurs

3.1 Travailler en projet

Un projet Cowork est un espace de travail associé à un dossier local sur votre PC. Il contient les instructions permanentes de Claude, les fichiers de référence, la mémoire accumulée et les tâches programmées.

📂

Dossier local lié

Claude peut lire le contenu du dossier, y écrire des résultats et y stocker sa mémoire de projet pour les itérations futures.

📋

Instructions permanentes

Rédigées une fois, disponibles à chaque exécution. Plus besoin de ré-expliquer le contexte à chaque nouvelle conversation.

🗄

Mémoire partagée

La mémoire est partagée entre toutes les discussions d'un même projet, mais non transférable d'un projet à l'autre.

Exemple d'instructions de projet "Analyse mon pipeline commercial sur HubSpot. Regarde les deals gagnés, perdus et les deals ouverts à fort potentiel de closing. Génère un rapport hebdomadaire et envoie-le sur Slack."

3.2 Choix du modèle et niveau d'effort

Modèle Usage recommandé Consommation tokens
claude-opus-4-8 Raisonnement complexe, analyse approfondie Elevée — à doser
claude-sonnet-4-6 Tâches courantes, tâches programmées récurrentes Modérée — recommandé
Attention à la consommation de tokens Le niveau d'effort "max" pousse le modèle à sa pleine capacité de réflexion mais multiplie la consommation de tokens. Pour une recherche simple (météo, actualités), l'effort minimal suffit. Réservez le niveau élevé aux problèmes complexes.

3.3 Programmer une tâche récurrente

1

Ouvrir la section "Schedule"

Depuis l'interface du projet, cliquer sur + dans la section "Programmé" pour créer une nouvelle tâche.

2

Nommer et décrire la tâche

Donner un nom explicite (ex. : "Hebdomadaire — Pipeline HubSpot") et décrire précisément l'action à exécuter. Cette description est le prompt d'exécution automatique.

3

Choisir le modèle et la fréquence

Sélectionner le modèle (Sonnet suffit pour la plupart des tâches récurrentes) et définir la fréquence : quotidienne, hebdomadaire, à une heure précise. Exemple : tous les lundis à 9h.

4

Activer les connecteurs nécessaires

Si la tâche doit lire HubSpot et envoyer sur Slack, ces deux connecteurs doivent être configurés et autorisés au préalable.

3.4 Connecteurs

Les connecteurs se configurent dans Settings → Customize → Connecteurs. Chaque connecteur expose des permissions granulaires que vous pouvez autoriser, bloquer ou soumettre à validation manuelle.

HubSpot / Salesforce Slack Notion Figma Miro Linear Semrush (SEO) Google Sheets + de 30 autres...
Sécurité des permissions Par défaut, préférez la validation manuelle ("Toujours demander approbation") pour chaque permission, surtout en phase d'apprentissage. L'option "Toujours autoriser" est pratique mais réduit votre visibilité sur les actions effectuées par Claude.

3.5 Skills et Plugins

📄

Skills (compétences)

Fichiers Markdown (.md) qui décrivent une méthodologie ou un ensemble de bonnes pratiques. Claude les consulte automatiquement selon le contexte. Structure : un fichier skill.md (sommaire) + des fichiers thématiques.

🔌

Plugins

Bundles développés par Anthropic et ses partenaires. Combinent un connecteur et un skill préconfiguré. Exemple : le plugin Figma inclut la connexion à l'API et les bonnes pratiques d'utilisation.

Analogie pédagogique Un skill, c'est un cours condensé que Claude peut consulter quand il en a besoin. Il lit la description du skill pour décider s'il est pertinent, puis plonge dans le contenu si nécessaire. Vous pouvez par exemple créer un skill "Expert CRM" ou "Directeur commercial" pour affiner la qualité des analyses.
04

Les live artifacts

Dashboards interactifs persistants

Un live artifact est une page HTML interactive, générée par Claude, qui persiste sur votre PC même après la fermeture de la session Cowork. Il se met à jour à la demande depuis les données sources (CRM, email, agenda...).

📊

Dashboard pipeline commercial

Affiche le nombre de deals, le win rate, l'open pipeline, les deals à clore prochainement. Chaque deal peut être un lien direct vers votre CRM.

📧

Suivi des emails en attente

Liste des messages sans réponse, triés par priorité ou ancienneté, connecté à votre messagerie.

✈

Planificateur de voyage

Comparatif de vols, hôtels et activités avec les prix en temps réel, exportable en Google Sheets ou Excel.

Créer un live artifact : étapes clés

1

Choisir le connecteur de données

Depuis l'interface Artifact, sélectionner la source (HubSpot pour le pipeline CRM, Gmail pour les emails, etc.).

2

Définir les métriques à afficher

Préciser à Claude les indicateurs souhaités. Plus la demande est précise, meilleur sera le résultat.

3

Demander la construction du dashboard

Exemple : "Build everything in a single dashboard". Claude génère la page HTML avec toutes les données intégrées.

4

Épingler et rafraîchir

L'artifact s'épingle dans l'interface pour un accès rapide. Le bouton "Reload" met à jour les données depuis la source en un clic.

Bon à savoir Vous pouvez continuer à discuter avec Claude directement depuis l'artifact ouvert pour l'améliorer, ajouter des sections ou modifier les visuels, sans repartir de zéro.
05

Claude in Chrome

Navigation autonome et travail à distance

L'extension Claude in Chrome permet à Claude de naviguer sur le web de façon autonome, ouvrir des onglets, lire des pages et extraire des données en temps réel — avec votre validation.

Installation

1

Activer dans les paramètres

Dans l'application desktop : Settings → Claude in Chrome Settings → Autoriser l'extension.

2

Installer depuis le Chrome Web Store

Rechercher "Claude" dans le Chrome Web Store, installer l'extension officielle. Elle apparaît ensuite dans la barre d'outils du navigateur.

3

Autoriser les actions au premier lancement

Lors de la première navigation automatique, Chrome demandera une validation. Cliquer "Autoriser" pour que Claude puisse opérer sur les onglets.

Cas d'usage illustré : préparer un voyage au Japon

Instructions données à Claude "Pour les billets d'avion, utilise Kayak.com. Pour les hôtels, regarde Booking.com. Pour les activités, choisis les sources les plus pertinentes. Génère un fichier comparatif avec les prix pour chaque option. Pose-moi les questions nécessaires avant de commencer."

Claude ouvre automatiquement les onglets Kayak et Booking, navigue sur les pages de résultats selon les dates et critères fournis, extrait les données et génère un comparatif structuré avec totaux par catégorie et budget global estimé.

Point de vigilance : prompt injection Lorsque Claude navigue sur le web de façon autonome, il peut tomber sur des pages contenant des instructions malveillantes cachées (prompt injection). Restez attentif aux permissions demandées lors de sessions de navigation automatique.

Travail à distance via Dispatch

La fonctionnalité Dispatch permet de piloter votre session Cowork depuis votre téléphone, tant que votre PC est allumé. Cela transforme votre ordinateur en un agent toujours disponible, operable depuis n'importe où.

📱

Connexion téléphone

Ouvrir l'application Claude sur mobile, scanner le QR code depuis Dispatch dans l'interface desktop. La connexion est instantanée.

💻

PC comme serveur d'agent

Votre ordinateur allumé devient un poste de travail distant. Claude peut continuer à naviguer, analyser et produire des résultats pendant que vous êtes en déplacement.

Récapitulatif

Ce qu'il faut retenir

🧩

Commencez par un projet ciblé

Choisissez une tâche récurrente précise et construisez un projet Cowork autour d'elle. Un projet CRM, un projet veille, un projet rapport hebdomadaire.

📝

Investissez dans les instructions

Plus vous détaillez le contexte, les outils à utiliser et le format de sortie attendu, plus les résultats seront précis. Traitez Claude comme un nouveau collaborateur.

🧠

Alimentez la mémoire

Notez ce qui fonctionne, ce qui ne fonctionne pas. La mémoire est partagée dans un projet, mais pas entre projets. Un projet = une expertise.

⚙

Utilisez les skills

Créez des fichiers Markdown décrivant les méthodes que Claude doit appliquer. Un skill "Expert CRM" ou "Analyste commercial" améliore sensiblement la qualité des sorties.

🔒

Gérez les permissions finement

Ne donnez pas "Toujours autoriser" par défaut. Préférez la validation manuelle au départ, puis assouplez selon la confiance acquise dans chaque flux.

🚀

Itérez et progresso

Cowork est un point d'entrée dans l'automatisation. Une fois à l'aise avec les projets et les agents, Claude Code devient la prochaine étape naturelle.

Métaphore clé Claude Cowork fonctionne comme un stagiaire intelligent à qui vous confiez des missions. Plus vous lui donnez de contexte, de méthodes de travail et de retours d'expérience, plus ses livraisons s'améliorent. La qualité de votre brief détermine la qualité de l'output.

Quiz — Claude Cowork : les fondamentaux

1. Quelle condition technique est indispensable pour utiliser Claude Cowork ?

Avoir la version gratuite de Claude et un navigateur Chrome
Télécharger l'application desktop Claude et disposer d'un abonnement payant
Avoir Claude Code installé au préalable

2. Quelle est la principale différence entre la mémoire d'un projet Cowork et la mémoire de Claude Chat ?

La mémoire de Chat est plus puissante car elle est partagée entre tous les projets
Il n'y a aucune différence, les deux systèmes fonctionnent de la même façon
La mémoire Cowork est persistante et partagée dans un projet, stockée dans un dossier local, non transférable entre projets

3. Qu'est-ce qu'un skill dans Claude Cowork ?

Un plugin payant développé par Anthropic pour connecter des outils tiers
Un fichier Markdown décrivant une méthodologie que Claude consulte selon le contexte
Un script Python exécuté automatiquement à chaque tâche planifiée

4. Quel risque de sécurité est spécifiquement mentionné pour Claude in Chrome ?

Le vol de mots de passe stockés dans Chrome
L'installation de logiciels malveillants sur le PC
Le prompt injection : des pages web contenant des instructions malveillantes que Claude pourrait exécuter involontairement

5. Qu'est-ce qu'un live artifact dans Cowork ?

Un fichier Excel généré automatiquement à chaque exécution de tâche
Une page HTML interactive persistante sur le PC, connectée à des données en temps réel, qui survit à la fermeture de la session
Un rapport PDF généré depuis HubSpot et envoyé par Slack

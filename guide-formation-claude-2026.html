01

Plans, tarifs et création de compte

Claude est accessible sur claude.ai avec plusieurs formules. Avant de commencer, il est utile de comprendre ce que chaque plan permet réellement.

Plan Usage Modèles accessibles Remarque
Gratuit Test et découverte Claude Sonnet 4.6, Haiku Limité en volume — pas pour un usage professionnel soutenu
Pro Usage individuel régulier Sonnet + Opus 4.6 Accès à Claude Code, projets avancés
Max 5x / 20x Utilisation intensive Tous les modèles Volume multiplié × 5 ou × 20 par rapport au Pro
Team / Enterprise Équipes et organisations Tous les modèles Projets partagés, mémoire collaborative

💡 Conseil formateur

Pensez Claude comme un collaborateur supplémentaire. Dans cette optique, le coût mensuel est très faible comparé à une ressource humaine. Commencez par le plan Pro, passez à Max si vous constatez une réelle contrainte de volume.

Créer son compte — pas à pas

1
Accéder à claude.ai

Cliquer sur « Essayer Claude ». Possibilité de s'inscrire par e-mail ou via Google.

2
Vérifier son numéro de téléphone

Un code de vérification est envoyé par SMS. Cette étape est obligatoire.

3
Choisir son usage

Personnel, avec une équipe, ou professionnel. Ce choix peut être modifié ultérieurement dans les paramètres.

4
Désactiver l'entraînement sur vos données

Important : désactiver l'option « Améliorer Claude avec mes conversations ». Cela protège la confidentialité de vos échanges professionnels.

5
Personnaliser son profil

Renseigner son rôle, ses centres d'intérêt, ses préférences. Ces informations sont utilisées pour adapter les réponses de Claude.

⚠️ Confidentialité

L'option « Améliorer Claude » est activée par défaut lors de certaines inscriptions. Vérifiez dans Paramètres → Confidentialité que cette option est bien désactivée, surtout si vous traitez des données professionnelles sensibles.

02

Interface et paramètres essentiels

L'interface de Claude est organisée en trois zones principales. Prendre le temps de la configurer correctement dès le départ évite de nombreuses frictions par la suite.

◧

Barre latérale gauche

Historique des conversations, artefacts, projets. Peut être masquée pour plus de concentration.

◫

Zone de chat centrale

Saisie des messages, sélection du modèle, ajout de fichiers, choix du style de rédaction.

◨

Panneau artefacts

S'ouvre à droite lorsque Claude génère du code ou un mini-outil. Affiche le rendu visuel ou le code source.

Paramètres clés à configurer

Paramètre Emplacement Recommandation
Préférences personnelles Paramètres → Général Renseigner son rôle, son contexte professionnel — améliore la pertinence de toutes les réponses
Mémoire Paramètres → Capacités Activer pour que Claude mémorise des informations entre les sessions
Chargement des outils Paramètres → Capacités Laisser sur « charger au besoin » pour ne pas consommer inutilement des tokens
Artefacts Paramètres → Visuels Activer. Option « artefacts alimentés par IA » à activer selon les besoins avancés
Exécution de code Paramètres → Visuels Activer pour exploiter la génération d'outils interactifs
Confidentialité localisation Paramètres → Confidentialité Selon usage : utile pour les requêtes locales, à désactiver pour la confidentialité maximale

ℹ️ Statut de la plateforme

En cas de dysfonctionnement, consultez status.anthropic.com pour vérifier l'état des serveurs en temps réel avant de chercher la cause côté utilisateur.

Choisir son modèle

Le sélecteur de modèle est accessible directement dans la zone de chat. Chaque modèle correspond à un compromis vitesse/puissance/coût.

Modèle Caractéristique Cas d'usage typique
claude-haiku Le plus rapide, le plus léger Réponses courtes, tâches simples, prototypage rapide
claude-sonnet-4.6 Équilibre optimal vitesse/qualité Usage quotidien général, rédaction, analyse, formation
claude-opus-4.6 Le plus puissant, plus lent Raisonnement complexe, code avancé, analyse approfondie
Opus 4.6 — 1M tokens Fenêtre contexte étendue Analyse de bases de code entières, documents longs

Modes d'interaction

⌨️

Mode texte standard

Saisie clavier avec support des fichiers, images, CSV. Le mode de base, toujours disponible.

🎤

Mode vocal

Conversation parlée bidirectionnelle. Idéal en déplacement, en voiture, ou pour tester des idées à voix haute.

👻

Mode incognito

Aucune mémorisation, aucune sauvegarde, aucun entraînement. Pour les échanges sensibles ponctuels.

🔍

Recherche web intégrée

Activée par défaut. Permet à Claude d'accéder à des informations récentes pour enrichir ses réponses.

💡 Raccourci clavier utile

Depuis l'application desktop, double-appui sur Option (Alt) ouvre une barre de commande rapide accessible depuis n'importe quelle application. Il est possible de faire glisser une capture d'écran directement dans cette barre.

03

L'art du prompting — méthode GCAO

La qualité d'une réponse dépend directement de la qualité du prompt. Un prompt générique produit une réponse générique — identique à ce que n'importe qui obtiendrait. La méthode GCAO structure chaque demande pour obtenir des résultats spécifiques, actionnables et personnalisés.

Goal + Context + Action + Output
Composant Définition Exemple concret
G — Goal
(Objectif)		 Quel résultat final visez-vous ? Soyez précis sur l'intention. « Mon objectif est de créer des vidéos YouTube qui performent bien pour ma chaîne. »
C — Context
(Contexte)		 Quelles informations Claude doit-il posséder pour comprendre votre situation ? Joignez des fichiers si pertinent. « Les analytics de mes vidéos passées sont joints en pièce jointe. »
A — Action
(Action)		 Quelle(s) tâche(s) précise(s) Claude doit-il exécuter ? « Analyse mes statistiques. Ensuite, recherche via web search les tendances de ma niche et propose 3 idées de vidéos. »
O — Output
(Format de sortie)		 Comment voulez-vous recevoir le résultat ? Format, longueur, structure. « Présente sous forme de liste, avec titre, durée, cible et une brève justification pour chaque idée. »

Prompt générique vs. Prompt GCAO — comparaison

❌ Prompt générique
Donne-moi des idées pour développer ma chaîne YouTube.
✅ Prompt GCAO structuré
OBJECTIF : Créer des vidéos YouTube qui performent bien pour ma chaîne.
CONTEXTE : Les analytics de mes vidéos passées sont joints en CSV.
ACTION : Analyse mes statistiques passées, puis effectue une recherche web
sur les tendances actuelles de ma niche. Propose 3 idées de vidéos basées
sur ce qui a fonctionné ET sur les tendances du moment.
FORMAT : Liste à puces avec, pour chaque idée : titre, durée estimée,
cible audience, et une courte justification.

🎯 Règle d'or

Le prompt générique reçoit des conseils qui conviendraient à n'importe quelle chaîne YouTube. Le prompt GCAO reçoit des recommandations basées sur vos données réelles, adaptées à votre cas précis. La différence est fondamentale.

Styles de rédaction personnalisés

Claude peut adopter un style de rédaction spécifique. Depuis la zone de chat, le menu de style permet de choisir parmi les styles prédéfinis (normal, concis, explicatif, formel) ou de créer un style personnalisé en fournissant des exemples de sa propre écriture.

💡 Créer son style maison

Dans « Créer et modifier les styles », il suffit de coller des exemples de textes que vous avez écrits — e-mails, rapports, comptes rendus. Claude analyse votre tonalité et réplique votre style dans ses réponses.

04

Projets — organiser son travail dans la durée

Une conversation classique est idéale pour une question ponctuelle. Mais pour un travail récurrent — développer une stratégie, suivre un projet client, préparer une formation — les Projets offrent une mémoire persistante, des fichiers de référence et des instructions permanentes.

📂

Mémoire silotée

Chaque projet possède sa propre mémoire, indépendante des autres conversations et projets.

📎

Fichiers en contexte permanent

Joindre des documents (CSV, PDF, notes) qui seront disponibles dans toutes les conversations du projet.

📋

Instructions persistantes

Rédiger une fois les instructions de comportement — objectif, rôle, contraintes — et les réutiliser à chaque session.

👥

Collaboration d'équipe

Partager un projet avec d'autres utilisateurs pour travailler collectivement dans le même contexte.

Structurer les instructions d'un projet avec GCAO

Les instructions de projet peuvent elles-mêmes être rédigées selon la méthode GCAO, ce qui évite de répéter le contexte à chaque nouvelle conversation.

Exemple d'instructions de projet
OBJECTIF DU PROJET :
Agir comme consultant RH pour aider à structurer un programme de
développement managérial pour une organisation de 150 collaborateurs.

CONTEXTE :
Les fichiers joints contiennent le diagnostic RH initial, les résultats
de l'enquête d'engagement et les profils des populations cibles.

ACTION (comportement par défaut) :
Poser les bonnes questions avant de prescrire. Agir en consultant,
pas en exécutant. Si une demande est vague, demander des précisions.

FORMAT :
Adapter la longueur des réponses à la longueur des messages reçus.
Privilégier un ton direct et professionnel. Utiliser des listes
uniquement lorsque c'est pertinent pour la clarté.

Mémoire et contexte entre sessions

En dehors des projets, Claude peut mémoriser des informations générales sur l'utilisateur (rôle, préférences, contexte) entre les conversations. Cette mémoire est gérée dans Paramètres → Confidentialité → Préférences de mémoire.

ℹ️ Importer sa mémoire depuis d'autres IA

Claude propose un mécanisme d'import des mémoires depuis ChatGPT (et d'autres). Un prompt spécifique est fourni à copier dans l'autre outil ; le résultat est ensuite collé dans Claude pour transférer le contexte accumulé.

05

Artefacts — créer des outils vivants

Un artefact est un mini-outil ou un document interactif généré par Claude et exécuté directement dans l'interface. C'est l'une des fonctionnalités les plus distinctives de Claude : il ne se contente pas de décrire un outil, il le construit et le rend opérationnel immédiatement.

🌐

Sites web et portfolios

Prototypes de sites, landing pages, portfolios. Le code HTML/CSS/JS est généré et rendu en temps réel.

🎮

Mini-applications

Calculateurs, convertisseurs, simulateurs, jeux. Des outils réutilisables créés en quelques minutes.

📊

Tableaux de bord

Visualisations de données, graphiques, analyses visuelles. À sauvegarder comme artefact pour y revenir.

📝

Documents structurés

Rapports, présentations, guides. Format plus riche que le simple texte de chat.

Créer un artefact depuis zéro

1
Nouveau chat → « Nouvel artefact »

Dans la barre latérale gauche, ou directement depuis le bouton d'artefacts.

2
Choisir le type

Application/site web, document, visualisation, outil… Claude guide avec des questions pour préciser le besoin.

3
Affiner itérativement

Décrire les modifications souhaitées en langage naturel. Claude modifie le code en préservant ce qui fonctionne déjà.

4
Publier et partager

« Publier et copier le lien » génère une URL partageable. Toute personne ayant le lien peut utiliser l'outil.

💡 Cas d'usage professionnel

Besoin d'un outil de conversion de fuseaux horaires avant un appel international ? Demandez à Claude de créer un artefact avec un curseur horaire. Vous l'aurez en deux minutes, utilisable immédiatement, sans télécharger aucune application.

Visualisations inline

Contrairement aux artefacts qui s'ouvrent dans un panneau séparé, les visualisations inline s'intègrent directement dans le fil de la conversation. Elles sont idéales pour explorer des données de façon conversationnelle — poser une question sur ses chiffres et voir un graphique apparaître au fil de l'échange.

Une visualisation inline peut être sauvegardée en artefact en la survolant et en cliquant sur l'icône de sauvegarde.

06

Claude Co-work — Claude sur votre ordinateur

Claude Co-work est disponible via l'application desktop (Mac, Windows). Il donne à Claude un accès à votre système de fichiers, vos applications et votre navigateur — transformant Claude d'un assistant conversationnel en un agent qui agit sur votre machine.

⚠️ Prérequis

Co-work nécessite l'application desktop Claude (à télécharger sur claude.ai) et les autorisations d'accessibilité et d'enregistrement d'écran accordées à l'application dans les paramètres système.

📁

Gestion de fichiers

Trier, déplacer, renommer, organiser des fichiers entre dossiers selon des règles que vous définissez.

⏱️

Tâches planifiées

Définir des automatisations récurrentes : nettoyage de bureau quotidien, rapports hebdomadaires, etc.

📱

Dispatch (mobile)

Piloter Co-work depuis son téléphone pendant que l'ordinateur est au bureau. Déléguer à distance.

🌐

Claude in Chrome

Extension Chrome qui permet à Claude de naviguer, cliquer, capturer des pages web pour vous.

Instructions globales Co-work

Dans Paramètres → Co-work → Instructions globales, il est possible de définir des préférences et conventions permanentes : comment Claude doit nommer les fichiers, dans quels dossiers déposer quoi, quel format utiliser pour les comptes rendus. Ces instructions s'appliquent à toutes les sessions Co-work.

Connecteurs (MCP — Model Context Protocol)

Les connecteurs permettent à Claude de lire et d'écrire dans des applications tierces. Le MCP (Model Context Protocol) est le standard technique qui rend cela possible.

Connecteur Ce que Claude peut faire
Gmail Lire, rédiger, envoyer des e-mails, créer des brouillons
Google Agenda Lire le planning, créer des événements, préparer des briefs journaliers
Notion Lire les bases de données, créer des pages, mettre à jour des CRM
Google Drive Accéder, lire, créer des fichiers dans Drive
HubSpot Interroger le CRM, enrichir les contacts, créer des tâches
GitHub Lire/modifier du code, créer des branches, commiter
Slack Envoyer des messages, lire des canaux, résumer des conversations

🔑 Cas d'usage combiné

Exemple concret : Claude lit un CRM Notion pour identifier les contacts non relancés depuis 60 jours, rédige un e-mail personnalisé pour chacun via Gmail, et crée les brouillons — le tout en une seule instruction. C'est la puissance de la combinaison connecteurs.

Compétences (Skills)

Une compétence (skill) est un processus automatisé et reproductible que Claude exécute de la même façon à chaque fois. Là où une conversation ordinaire peut donner des résultats variables, une compétence impose un cadre précis : les mêmes étapes, les mêmes outils, le même format de sortie.

A
Créer par description

Expliquer à Claude ce qu'on veut automatiser. Il pose des questions et construit le fichier de compétence.

B
Rétro-ingénierie d'un résultat existant

Obtenir un bon résultat en chat classique, puis dire à Claude : « Crée une compétence qui fait exactement ce qu'on vient de faire. »

C
Importer une compétence tierce

Télécharger un fichier .zip de compétence existant et l'uploader dans Paramètres → Compétences.

💡 Activer une compétence

Dans Co-work, les compétences s'invoquent avec une commande slash. Exemple : taper /cook lance la compétence de génération de recettes. Taper /daily lance le brief journalier depuis l'agenda. Ces commandes sont définies lors de la création de la compétence.

07

Claude Code — développement assisté par IA

Claude Code est l'environnement de développement intégré à Claude. Il permet de créer, modifier, tester et déployer du code directement depuis l'interface Claude, sans basculer entre un éditeur et une interface IA.

⚠️ Niveau technique requis

Cette section implique des concepts de développement (terminal, dépôts Git, variables d'environnement, déploiement). Elle est accessible aux non-développeurs mais nécessite d'être à l'aise avec la logique technique. En cas de blocage, Claude lui-même est le meilleur recours pour demander des explications.

💻

Interface visuelle

Onglet « Code » dans l'application desktop. Interface graphique avec prévisualisation intégrée.

⬛

Terminal

Commande claude dans n'importe quel terminal. Plus flexible, préféré des développeurs.

☁️

Environnement cloud

Anthropic héberge l'exécution sur ses serveurs. Opérationnel depuis n'importe où, y compris depuis un téléphone.

🔗

Connexion SSH

Se connecter à un serveur distant et opérer sur une base de code hébergée à distance.

Paramètres importants de Claude Code

Paramètre Valeurs Impact
Effort Faible / Moyen / Élevé / Maximum Détermine le nombre d'agents et la profondeur d'analyse. Maximum pour les grandes bases de code.
Permissions Demander / Auto-accepter / Plan / Bypass Niveau d'autonomie accordé à Claude. Recommandé : « demander » pour les opérations sensibles.
Modèle Opus 4.6 / Opus 4.6 1M / Sonnet Opus 4.6 1M pour les très grandes bases de code nécessitant un contexte étendu.

Le fichier CLAUDE.md

Au sein d'un répertoire de projet, un fichier CLAUDE.md fournit à Claude des instructions permanentes sur ce projet : structure des dossiers, conventions de code, objectifs, contraintes. C'est l'équivalent des instructions de projet, mais pour le code.

Exemple de CLAUDE.md
# Mon projet personnel

## Structure
Répertoire racine pour tous les sous-projets tutoriels.
Chaque sous-dossier = un projet distinct.

## Conventions
- Toujours créer un .gitignore avant le premier commit
- Nommer les branches : feature/nom-fonctionnalite
- Ne jamais commiter de clés API ou secrets

Commandes utiles en terminal

  • /model — Changer de modèle en cours de session
  • /effort — Ajuster le niveau d'effort (low / medium / high)
  • /btw votre question — Poser une question latérale sans interrompre la tâche en cours
  • /plan — Demander à Claude de rédiger un plan avant d'agir
  • /loop — Exécuter une tâche en boucle automatisée (cron)
  • /schedule — Programmer une tâche à une heure donnée

Déploiement avec Netlify

Claude Code intègre un connecteur Netlify qui permet de déployer un site web directement depuis une session de code. Le flux typique est le suivant :

1
Créer le site localement avec Claude

Générer les fichiers du site dans un répertoire de projet via Claude Code.

2
Connecter le connecteur Netlify

Dans Co-work ou Claude Code → Options → Connecteurs → Netlify → Autoriser.

3
Déployer

Dire à Claude : « Déploie ce site sur Netlify. » Il exécute la commande et retourne l'URL live.

4
Connecter à GitHub pour le déploiement continu

Lier le repository GitHub au projet Netlify. Chaque push sur la branche principale redéploie automatiquement le site.

5
Configurer les variables d'environnement

Ajouter les clés API dans Netlify → Configuration du projet → Variables d'environnement. Ces clés ne sont jamais exposées dans le code.

Travailler depuis son téléphone

Grâce aux environnements cloud, Claude Code fonctionne depuis n'importe quel appareil connecté à Internet. Le code s'exécute sur les serveurs d'Anthropic, pas sur le téléphone.

🎯 Ce que cela signifie concrètement

Depuis votre téléphone, vous pouvez modifier le contenu d'un site web, publier un article de blog, corriger un bug dans votre application, planifier des tâches automatisées. Tout cela en chatant en langage naturel, sans toucher à un éditeur de code.

Récapitulatif — ce que vous savez faire maintenant

À l'issue de cette formation, les participants maîtrisent les éléments suivants :

  • Créer et configurer un compte Claude avec les bons paramètres de confidentialité

  • Structurer un prompt efficace avec la méthode GCAO
  • Créer et utiliser des projets avec fichiers et instructions persistantes
  • Générer des artefacts interactifs (outils, sites, tableaux de bord)
  • Utiliser Co-work pour automatiser des tâches sur son ordinateur
  • Connecter des applications tierces via les connecteurs MCP
  • Créer des compétences (skills) pour des processus reproductibles
  • Utiliser Claude Code pour développer et déployer des projets web
  • Opérer Claude depuis son téléphone via les environnements cloud

💡 Prochaine étape recommandée

Identifiez dans votre activité professionnelle une tâche répétitive que vous faites au moins 2 fois par semaine. Créez un projet Claude ou une compétence dédiée à cette tâche. C'est le chemin le plus court vers un gain de temps concret et mesurable.
